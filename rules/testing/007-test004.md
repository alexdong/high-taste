# Avoid logic or loops in tests—keep them simple.

**ID**: TEST004  
**Category**: Unit Tests → testing

## Rule Description
Avoid logic or loops in tests—keep them simple.

## Before
```python
for i in range(10):
    assert add(i, i) == 2*i
```

## After  
```python
@pytest.mark.parametrize('x', range(10))
def test_addition(x):
    assert add(x,x) == 2*x
```
